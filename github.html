Merge Conflicts

Cuando se trabaja un proyecto hay varias personas que trabajan los códigos

Que pasa si ambas peersonas hacen cambios al mismo sector de códigos y luego tratar de sincronizar juntos?

ambos están realizando cambios. A Estipo e Conflicto se llama MERGE CONFLICT

Nos estamos involucrando en una situación en la que GIT no sabe que hacer

Si utilizo GIT PULL se produce algún tipo de conflicto de lo que está ON LINE con los Commits

Vamos a tener mensajes como: CONFLICT etcetera

Van aparecer mensajes como: a=1 ========= <<<<>>468798725654

Esta información la vamos a filtrar de esta manera:

Everything in between these arrows at the top and equal signs are MY CHANGES the changes I have made on my version of the repository that are somehow conflicting with some other changes.

Everything after these equal signs and these arrows down here are the remote changes, the changes from GitHub that I'm trying to pull in that and somehow are conflicting with what I've currently been working on. That long number is the conflicting commit

that is likely to be unique that helps to identify any particular commit, y lo genera automáticamente cada vez que se hace commit

Para solucionar este conflicto merge tenemos que remover todos estos marcadores de conflictos que se encuentra en este file de texto y decidir qué queremos hacer para solucionar el conflicto

Se trata de remover todas las flechas, los iguales y los espacios
